1. PROJE ÖZETİ
================================================================================

Game Lens, oyun içi altyazıları gerçek zamanlı yakalayıp hedef dile çeviren
ve oyunun üzerine overlay olarak gösteren bir masaüstü uygulamasıdır.

Mevcut teknoloji stack'i:
  - Python 3.11
  - Tkinter (UI)
  - RapidOCR + ONNX (OCR)
  - MSS (ekran yakalama)
  - OpenCV (görüntü ön işleme)
  - Argos Translate + ctranslate2 (offline çeviri, GPU/CUDA)
  - deep-translator (Google Translate fallback)

Mevcut mimari (Adapter Pattern):
  - src/core/engine.py          → Pipeline orkestrasyonu
  - src/core/hook_manager.py    → Capture > Preprocess > OCR > TemporalFilter > Dispatch
  - src/core/temporal_filter.py → Altyazı/tabela/HUD/gürültü ayrıştırma
  - src/core/translator.py      → Argos Translate + GPU + cache
  - src/core/vision.py          → MSS ekran yakalama
  - src/core/preprocessor.py    → Oyun başına OCR profilleri
  - src/core/text_cleaner.py    → Slang/geniş zaman normalizasyonu
  - src/core/validator.py       → OCR gürültü filtreleme
  - src/core/manual_translations.py → Fuzzy match çeviri hafızası
  - src/adapters/               → Metro 2033, Firewatch, RDR2, Mock adapter'ları
  - src/ui/launcher.py          → Oyun seçici UI
  - src/ui/overlay.py           → Şeffaf overlay penceresi

2. ÜRÜN VİZYONU — İKİ FAZLI YAKLAŞIM
================================================================================

FAZ 1: EĞİTİM + DATASET OLUŞTURMA
──────────────────────────────────

  Oyun Açık (OCR + TemporalFilter)
           │
           ▼
      Her "emit" frame → otomatik kaydet
           │
           ├─ frame.png (altyazı olan ekran görüntüsü)
           ├─ bbox: [x, y, w, h] (TemporalFilter'ın bildiği konum)
           └─ text: "Watch out!" (OCR çıktısı)
           │
           ▼
      1000 frame birikince → YOLOv8-nano eğit
           │                         ↓
           │                   ~3MB .onnx model
           │
           ▼
      Tüm OCR metinleri → GPT/Claude API ile kaliteli çeviri
           │                         ↓
           │                   game_xx.pack (~2-5MB)
           │
           ▼
      Model + Dataset → GamePack yükle (CDN)

  İlk oyuncu bir oyunu ~2-3 saat oynadığında pack hazır hale gelir.
  2-3 kişi aynı oyunu oynarsa 1 saatte pack çıkar.
  Yeni çıkan bir oyun için aynı gün pack yayınlamak mümkündür.


FAZ 2: SON KULLANICI DENEYİMİ
──────────────────────────────

  Oyun başlat → GamePack var mı?
           │
           ├─ YOK → OCR + TemporalFilter (Faz 1'e katkı, ~200ms gecikme)
           │
           └─ VAR → Model yükle (.onnx)
                      │
                      ▼
                Ekran → Model inference (GPU 2ms / CPU 10ms)
                      │
                      ▼
                Altyazı bölgesi crop → OCR (30ms)
                      │
                      ▼
                Hash lookup → pack'ten çeviri (0ms)
                      │
                      ▼
                Overlay göster

                Toplam gecikme: ~40ms (insan gözüyle algılanamaz)

  Pack olmayan durumda da sistem çalışır:
    - OCR + canlı çeviri (Argos, ~200-400ms)
    - Çeviri cache'i zamanla dolar, hit rate %90+ seviyesine ulaşır
    - Kullanıcı oynadıkça cache kendi kendini iyileştirir
    - Bu oturumlardan toplanan veriler pack oluşturmak için kullanılır

EKSIKLER — YAPILACAKLAR
───────────────────────

  1. FRAME COLLECTOR
     TemporalFilter'ın "emit" kararı verdiği frame'leri otomatik
     kaydeden modül. Diske frame.png + bbox + OCR metni yazar.
     Tahmini süre: 2-3 saat.

  2. MODEL TRAINER
     YOLOv8-nano fine-tune pipeline'ı. 1000 frame birikince
     otomatik tetiklenir, ONNX model çıktısı üretir.
     Tahmini süre: 1-2 gün (Ultralytics API).

  3. ONNX INFERENCE RUNNER
     ONNX Runtime ile modeli runtime'da çalıştıracak sınıf.
     GPU'da 2ms, CPU'da 10ms hedef.
     Tahmini süre: 1 gün.

  4. GAMEPACK FORMATI
     .gamepack = model.onnx + dataset.json + metadata.json
     Tek bir zip/arşiv, 5-8 MB.
     Tahmini süre: 4-6 saat.

  5. AI ÇEVIRI BATCH
     Toplu OCR metinlerini GPT-4/Claude API ile bağlamsal
     kalitede çeviren pipeline.
     Tahmini süre: 1-2 gün.

  6. PACK SUNUCUSU
     Basit bir CDN/API endpoint. Oyun ID'si ile pack sorgula,
     yoksa upload et, varsa indir.
     Tahmini süre: 2-3 gün (Cloudflare R2 + basit API).

  7. KALIBRASYON SIHIRBAZI
     Kullanıcının fareyle altyazı bölgesini seçtiği, sistemin
     renk analizi yapıp OCR profili önerdiği interaktif UI.
     Tahmini süre: 2-3 gün.

  8. SUBTITLE PARSER'LAR
     Popüler oyunların altyazı dosyalarını doğrudan okuyan
     parser'lar (.locres, .xml, .txt, .yarn, .json).
     Tahmini süre: format başına 1-6 saat.

6. RİSKLER VE ÖNLEMLER
================================================================================

  1. STEAM ONAYI
     Overlay tabanlı tool'lar Steam'de geçmişte sorun yaşadı.
     → Steam'in overlays and hooks politikasını hukuki olarak araştır.
     → Alternatif olarak kendi web sitenden satıp Steam key dağıtabilirsin.

  2. ANTI-CHEAT FALSE POSITIVE
     EAC, BattlEye, VAC ekran yakalama/overlay'i cheat olarak algılayabilir.
     → Sadece single-player oyunları destekle, anti-cheat'li oyunlarda
       otomatik devre dışı kal.
     → WDA_EXCLUDEFROMCAPTURE ve WS_EX_TRANSPARENT bayrakları zaten var.

  3. DESTEK YÜKÜ
     300K kullanıcı = günde 500-1000 destek talebi.
     → Discord topluluğu + bilgi tabanı + gönüllü moderatörler.
     → Otomatik hata raporlama ve crash dump toplama.

  4. HER OYUN İÇİN AYRI PROFIL GEREKMESI
     → Kalibrasyon sihirbazı + otomatik veri toplama ile kullanıcı
       tarafında çözülür.
     → Topluluk pack'leri paylaştıkça kapsama organik büyür.

  5. YASAL GRİ ALAN
     Oyun ekranını yakalayıp üzerine overlay basmak EULA'da gri bölge.
     → Çeviri amaçlı overlay = fair use / accessibility tool olarak
       konumlandır.
     → Oyun dosyalarını modifiye etmiyorsun, sadece okuyorsun.

7. SONUÇ
================================================================================

Game Lens, teknik olarak uygulanabilir, pazar olarak karşılığı olan ve
doğru fiyatlandırmayla ($3-5) global ölçekte anlamlı gelir üretebilecek
bir üründür.

Güçlü yanlar:
  - Hibrit yaklaşım (pre-load + OCR fallback) ile geniş oyun kapsaması
  - Topluluk katkısıyla kendi kendini iyileştiren veri modeli
  - GPU hızlandırmalı offline çeviri, internet bağımsız
  - 40ms gecikme hedefi (pack varken), 200ms (pack yokken)
  - Rakipsiz bir alan, first-mover avantajı

Zayıf yanlar:
  - İlk sürüme kadar ciddi geliştirme gerekiyor (6-12 ay)
  - Her oyun için pack oluşturma süreci emek yoğun
  - Steam onayı ve anti-cheat riskleri netleşmeli
  - Teknik destek yükü kullanıcı sayısıyla lineer artar

Önerilen strateji:
  1. MVP'yi tamamla (ilk 3 madde: frame collector + trainer + inference)
  2. 5-10 popüler oyunla Steam'de Erken Erişim'e çık
  3. Topluluğu Discord'da büyüt, pack katkısını teşvik et
  4. Premium abonelik ile sürdürülebilir gelir modeline geç
  5. B2B lisanslama için stüdyolarla görüşmelere başla
